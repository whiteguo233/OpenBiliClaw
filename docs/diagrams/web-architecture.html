Web HTML 模块架构与流程图

`src/openbiliclaw/web/` 承载两套静态 HTML 前端：桌面端 `/web` 与移动端 `/m`。它们只调用本地 FastAPI，不采集平台行为，也不负责 Cookie 同步。

1. 架构图：FastAPI 托管的双 Web 入口

Static Surfaces

桌面端 Web `/web` mount 到 `web/desktop/`，`index.html + assets/js/app.js + assets/css/app.css`，大屏推荐首页。
移动端 Web `/m` mount 到 `web/`，`index.html + js/* + css/app.css`，手机端推荐、画像、对话、消息。
根路径跳转 `/` 302 到 `/web`；手机入口由插件二维码生成 `/m/` URL。

Frontend Runtime

移动端模块化 SPA `app.js` 路由和 shell，`state.js` 集中状态，`api.js` 请求，`stream.js` WebSocket，`views/*` 渲染各 tab。
桌面端单页控制器 `desktop/assets/js/app.js` 管理推荐、画像、聊天、设置、消息、delight 队列与本地 endpoint 设置。
runtime-stream 合并刷新 `activity.added / profile_updated / refresh.pool_updated / interest.probe / avoidance.probe` 进入 debounce / single-flight。

Backend API

推荐 API `/api/recommendations`、`/reshuffle`、`/append`、`/refresh`、`/recommendation-click`、`/feedback`。
画像 / 聊天 / 探针 `/profile-summary`、`/chat/turns`、`/interest-probes/*`、`/avoidance-probes/*`。
配置与资源代理 `/config`、`/runtime-status`、`/image-proxy`。图片代理做白名单、redirect、类型和 10MB 校验。

2. 前端模块边界

Shell / routing

移动端 hash tab 默认进入 `recommend`；桌面端通过 nav / drawer 切换首页、画像、聊天、设置。

State model

移动端 `state.js` 用订阅模式统一 `recommendations / profile / messages / runtimeStatus / chatTurns`。

Stream client

WebSocket 只负责接收 runtime events；断线后按退避重连，页面仍保留最近一次 REST 状态。

View models

`view-models.js` 把后端推荐、池子状态、平台标签和反馈 payload 规范化，DOM 渲染前先清洗。

Desktop controller

桌面端把 endpoint、本地存储、设置表单、delight 内聊、watch-later 和推荐卡片动作放在单控制器中。

Boundaries

Web UI 不读取平台 Cookie、不执行 XHS/DY/YT 任务、不采集站内行为；这些仍归浏览器插件。

3. 流程图：页面启动与实时同步

1 StaticFiles 返回 HTML

FastAPI 挂载 `/web` 和 `/m`；浏览器加载 CSS 与 JS。

2 初始化 shell

渲染导航、状态栏、空态和当前 tab 容器。

3 首屏 REST hydration

先取推荐列表；runtime status、activity、delight、profile 并行或异步补齐。

4 连接 runtime-stream

建立 WebSocket 后接收池子、探针、画像、活动和降级事件。

5 合并状态

`refresh.pool_updated` 只更新池子数字和 header，不覆盖用户 append 出来的推荐列表。

6 局部重渲染

推荐卡片、消息、画像、聊天 pending turn 按当前状态增量刷新。

4. 流程图：核心用户动作

推荐流

1 换一批 / 续页

主动 reshuffle 替换当前列表；append 传 `excluded_bvids`，保留已看的历史卡片。

2 打开推荐

先 POST `/recommendation-click`，再打开 `content_url`；封面统一走 `/image-proxy`。

3 反馈 / 稍后再看

`like / dislike / dismiss` 调反馈 API；watch-later 调本地书签端点并做乐观 UI。

画像、聊天、探针

1 读取画像摘要

`/profile-summary` 暴露五层画像、认知更新、正向兴趣和避雷候选。

2 Durable chat

提交 `/chat/turns` 后轮询 turn 状态；reload 后按 session / scope 恢复 pending 与历史。

3 Probe respond

普通兴趣、挑战兴趣和避雷探针走不同文案，但都调用后端 respond API 写回真实语义。

5. 降级与边界

后端离线 Web UI 显示连接失败 / 缓存配置；不会伪造默认配置或继续提交不可达请求。
推荐慢请求 移动端首屏有超时兜底，慢请求不会让页面一直停在 loading。
权限边界 Cookie 同步、平台页面任务和行为采集都必须由插件执行，Web 端只作为本地 API 客户端。
