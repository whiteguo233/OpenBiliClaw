1. 架构图：行为信号到五层画像

Inputs

跨平台事件 Bilibili / 小红书 / 抖音 / YouTube / Web 的 `view`、`favorite`、`like`、`follow`、`search`。
推荐反馈与点击 `recommendation_click`、`like`、`dislike`、`comment`、`dismiss`，带 `content_id / source_platform`。
Durable chat turns `scope=chat / delight / probe / avoidance_probe`，完成后进入受控学习链路。

Soul Core

SoulEngine 统一编排 `analyze_events()`、`build_initial_profile()`、`learn_from_dialogue()`、即时 cognition card 和画像读取。
ProfileUpdatePipeline 把新增 `ProfileSignal` 分层汇聚；空闲 tick 会驱动 speculator / avoidance lifecycle，不阻塞 API。
PreferenceAnalyzer 过滤 quick-exit，保留明确负反馈，输出偏好、避雷、推测兴趣 seed。
ProfileBuilder 基于 history + preference + awareness + insight 生成 `OnionProfile`。
Awareness / Insight 半日节流，把近期行为变成观察笔记和可验证假设。
DialogueInsightAnalyzer 从聊天中提取 goal / value / interest / dislike / state 候选信号。

Outputs

Memory layers `events`、`preference`、`awareness`、`insight`、`soul`，并镜像到画像 JSON / Markdown。
Interest / Avoidance probes 正向探针验证潜在兴趣；避雷探针只在确认后写入 `disliked_topics` 并清候选池。
Downstream consumers Discovery、Recommendation、Web / 插件画像页、OpenClaw skill 都读取同一份 profile summary。

2. 五层 OnionProfile

Core

核心特质、深层需求、MBTI。变更慢，要求强证据和 diff protection。

Values

价值观、驱动力、长期追求。每周期有限 add/remove，避免画像漂移。

Interest

领域树、specifics、喜欢与不喜欢方向、UP / 作者偏好。

Role

生活阶段、当前任务、正在经历的上下文。适合中频更新。

Surface

语气、内容形态、探索开放度和近期状态。更贴近当前体验。

3. 流程图：初始化画像

1 收集首轮信号

B 站 API + XHS / 抖音 / YouTube 插件 bootstrap task，统一转成事件。

2 写入事件层

去重、补 `source_platform`，保留原始 task-result 供排障。

3 偏好分析

`PreferenceAnalyzer` 提取 likes / dislikes / source mix / speculative seeds。

4 画像构建

`ProfileBuilder` 输出五层 `OnionProfile`，兼容旧 flat schema。

5 启动探针

`force_tick()` 立即生成正向与避雷候选，受 active quota 约束。

6 下游可读

`/api/profile-summary`、推荐表达、发现 prompt、OpenClaw 共享使用。

4. 流程图：增量学习与探针回写

行为 / 反馈增量

1 新事件进入 memory

账号同步、推荐点击、显式反馈、插件 task-result 都会变成事件。

2 即时与批量并行

单条 `dislike/comment` 生成 cognition card；批量达到阈值后重跑偏好分析。

3 CognitionCycle

awareness 失败不推进节流时间；下次 tick 可以马上重试。

4 LayerUpdaters

Interest / Surface 更快更新，Role / Values / Core 需要更强证据。

探针闭环

1 InterestSpeculator

生成 `near / lateral / bridge / wildcard`，普通池和挑战池分配额。

2 用户确认或聊天

强正向直接确认，弱正向先进 `short_term_exploration_buffer`。

3 AvoidanceSpeculator

只消费高确信负向信号；被动 quick-exit 不自动写长期避雷。

4 写回与清池

`apply_new_dislikes()` 写 `preference.disliked_topics`，同步 `soul` layer 并 purge pool。
