1. 架构图：Discovery 填池，Recommendation 独立消费

Pool Inputs

content_cache `pool_status='fresh'` 的候选来自 B 站策略、XHS / 抖音 / YouTube producer、通用 Web。
Readiness gates 可换候选必须具备分类、推荐文案、可跳转 URL，且不在近期已看窗口。
count_pool_readiness() 对外拆分 `available / raw / pending`，前端不会把 pending 误显示为可换。

Recommendation Core

RecommendationEngine.serve() `generate_recommendations()`、`reshuffle_recommendations()`、`append_recommendations()` 都委托到统一入口。
classify_pool_backlog() 未分类跨源候选补 `style_key / topic_group / relevance_score`，并带近期 negative exemplars。
precompute_pool_copy() 后台并发生成 `pool_expression / pool_topic_label`；遇到限流保留空值等下轮重试。
PoolCurator + MMR 五维 rec_score、topic supergroup、embedding MMR、topic / style / broad-topic cap 共同控制同批丰富度。

Outputs

recommendations 批量写入推荐历史，`content_cache` 对应行标记 `shown`。
Web / 插件 / OpenClaw `/api/recommendations`、`reshuffle`、`append` 和主动 delight 共享同一候选池语义。
Feedback loop 点击、喜欢、不喜欢、评论、忽略进入事件层；避雷确认会写 `disliked_topics` 并清池。

2. 流程图：serve() 热路径

1 读取库存口径

`available / raw / pending` 用于日志和 UI 诊断，避免 count/load 漂移被隐藏。

2 加载候选窗口

`get_pool_candidates()` 应用 fresh、分类、文案、URL、近期已看和 XHS self-author gates。

3 排除当前页

`append` 传入 `excluded_bvids`，防止续页重复展示。

4 合并近义主题

`_merge_topic_supergroups()` 用 embedding 把近义 topic_group 合成同一桶。

5 二次评分

`PoolCurator` 叠加 freshness、topic fatigue、source monotony、serendipity、feedback。

6 MMR 选择

优先使用预热 embedding；无 embedding 时走字符串 cap 和逐步回填。

7 写历史并返回

插入 `recommendations`，标记池项 `shown`，返回 `list[Recommendation]`。

3. 多样性与保护机制

Source-agnostic classification

推荐层只看内容特征，不用平台来源兜底做多样性 token；所有来源先补同一套 `style_key / topic_group`。

Precompute-first copy

`reshuffle / append` 默认只消费池内预生成文案，前端换一批不现场等待 LLM。

MMR embedding warm

discovery 入池、分类补评估、refresh tick 和冷启动都会 best-effort 预热 L2 embedding cache。

Amplification guard

刚确认的新兴趣有 24h rolling budget，同批命中同一新方向最多 `floor(limit * 0.25)`。

4. 流程图：反馈、点击与推荐池回流

推荐交互

1 打开内容

`/api/recommendation-click` 优先使用 payload 的 `content_url / source_platform`，避免跨源 ID 被套成 B 站 URL。

2 写点击事件

点击进入 events 和强画像信号，可被 Soul 和后续推荐语气消费。

3 推荐历史可追踪

推荐记录保留 `presented / feedback_type / feedback_note / feedback_at`。

反馈语义

1 like / comment

进入偏好与短期探索 buffer；必要时生成 cognition update。

2 dislike

作为强负向样本，进入 negative exemplars、PreferenceAnalyzer 和池子避让。

3 dismiss

软移除：标记候选 `feedbacked`，不更新画像，不扩大成话题级负反馈。
