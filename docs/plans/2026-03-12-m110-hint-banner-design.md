# M110 Popup 提示条增强设计

## 目标
提升 popup 底部提示栏的可感知度，让“这句记下了”“正在补货”“这次没刷出来”等即时反馈更容易被看见。

## 现状问题
- 当前提示栏是弱对比页脚文案
- 成功、提示、错误三类状态视觉上几乎没有区别
- 用户完成反馈后，很容易忽略系统是否真的记住了这次操作

## 方案
保留底部提示栏位置，不改整体布局，只增强提示条本身：
- 增加 `info / success / error` 三种状态
- 提示条采用轻胶囊/卡片式底色，而不是纯文字
- 左侧加入小状态点，强化“刚发生了什么”
- 新消息出现时只做很轻的高亮，不引入 toast 或悬浮层

## 实现边界
- 不改 popup 的信息架构
- 不新增通知中心
- 不影响推荐卡片、画像、聊天逻辑

## 测试
- 校验 footer 样式块存在新的状态样式
- 校验 JS 能根据提示类型切换 `data-tone`
